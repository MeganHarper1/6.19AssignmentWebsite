Waterfall Methodology

Overview and The Stages

The Waterfall Model was the first Process Model to be introduced. It illustrated the software developent process in a linear flow, meaning one phase cannot start until the previous phase in comlpete, the phases do not overlap. The 6 stages are:

  1. Requirement Gathering and analysis - All possible requirements for the system are stated and documented in a requirement specification legal binding document.
  2. Design - Based on th requirements from phase 1 the system is designed.
  3. Implementation - The system is first developed in small units.
  4. Integration and testing - All of the previously designed units are integreted to be together as 1 system after each unit being individually tested.
  5. Deployment - The system is deployed to the customer environment.
  6. Maintenance- Any issues that arise are fixed normally by releasing patches of software.
